# 진입 흐름 회귀 검증

## 목적

- `requestMappings` 기반 진입 흐름이 샘플에서 안정적으로 보이는지 확인한다.
- 리포트의 `Start Here`와 `Entry Flow`가 실제 진입 중심 구조를 보여주는지 확인한다.

## 체크 항목

- `npm run check`
- `npm test`
- `npm run analyze -- samples/legacy-java-ee-minimal`
- 샘플 스냅샷에서 controller 메타데이터에 `/sample/list.as` 존재
- 샘플 리포트에 `Primary Entry Flows` 존재
- 샘플 리포트에 `/sample/list.as` 존재
- 샘플 리포트에 `*.do` 존재
- 샘플 리포트에 `dispatcher-servlet.xml` 존재
- 샘플 리포트에 `Dispatcher` 존재
- 샘플 리포트에 `Spring Config` 존재
- 샘플 리포트에 `Open Data Flow` 존재
- 샘플 리포트에 `sampleService`, `sampleDao` 존재

## 추가 확인 포인트

- `가로형 / 카드형` 토글 존재
- `KR / EN` 토글 존재
- 탭 전환 시 전체 재렌더가 아닌 현재 탭 중심 동작

## 실제 프로젝트 리스크

- 특정 대형 레거시 프로젝트는 현재 실행 환경 제약 때문에 재생성 검증이 제한될 수 있다.
- 이 경우 기존 스냅샷 구조 확인과 로컬 샘플 회귀 검증을 기본 안전망으로 사용한다.
