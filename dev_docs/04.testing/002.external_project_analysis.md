# 외부 프로젝트 분석 검증 이력

## 검증 목적

- 분석 대상 프로젝트가 code2me 작업공간 밖에 있어도 실행 가능해야 한다.
- 결과 파일은 원본 저장소가 아니라 code2me 내부 저장소에 생성되어야 한다.

## 검증 항목

- `npm run check`
- `npm test`
- `npm run analyze -- samples/legacy-java-ee-minimal`
- `npm run analyze -- /path/to/reference-legacy-project`

## 당시 기대 결과

- 외부 경로 분석 시 `EROFS`, `EACCES` 없이 완료
- CLI 출력의 `outputDir`가 `.code2me-data/projects/...` 경로를 가리킴
- `summary.md`, `report.html`이 내부 저장소에 생성
- 원본 기준 프로젝트 경로 아래에는 `.code2me` 디렉터리가 생기지 않음

## 현재 상태

- 현재 구현은 사용자 요청에 따라 다시 `대상 프로젝트 아래 .code2me/` 저장 구조를 사용한다.
- 분석 리포트는 `Start Here`와 `Entry Flow` 중심으로 진입 흐름을 먼저 보여준다.
- 따라서 외부 읽기 전용 경로에서는 쓰기 실패가 발생할 수 있다.
