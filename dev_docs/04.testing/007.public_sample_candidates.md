# 공개 샘플 후보 목록

## 목적

- 합성 fixture 밖의 구조 다양성을 낮은 비용으로 확인할 공개 샘플 후보를 정리한다.
- 후보는 일상 회귀용이 아니라 `release gate` 또는 수동 품질 점검용으로 사용한다.

## 선정 기준

- 공개 접근 가능
- `web.xml`, Spring XML, JSP, mapper 등 현재 프로파일 경계와 맞닿아 있음
- 실패해도 어떤 축이 부족한지 해석 가능한 구조여야 함

## 후보

### 1. `mybatis/jpetstore-6`

- 이유:
  - `web.xml`, `applicationContext.xml`, JSP, MyBatis 구조를 함께 가진 공개 샘플
  - Spring MVC가 아니라 Stripes 계열 action 구조가 섞여 있어 profile boundary 점검에 유용
- 기대:
  - entry / JSP / mapper 탐지는 가능
  - handler/action 복원은 soft fail 가능성 높음
- 상태:
  - 2026-04-13 1차 게이트 수행

### 2. `spring-petclinic/spring-framework-petclinic`

- 이유:
  - JSP 기반 공개 샘플로 화면 흐름 확인에 좋음
  - Java config 중심이라 현재 detection 경계 확인용으로 적합
- 기대:
  - 현재 `legacy-java-ee`에서는 no-match 또는 boundary fail 가능성 높음
- 상태:
  - 2026-04-13 boundary check 수행

### 3. Struts / Stripes action-family 공개 샘플

- 이유:
  - 현재 프로파일이 잘못 잡아낼 수 있는 인접 프레임워크 경계를 확인하기 좋음
- 기대:
  - detection boundary 또는 mapping soft fail 기록 축적
- 상태:
  - `apache/struts-examples` `spring-struts`: 2026-04-13 hard fail 기록
  - `apache/struts-examples` `form-validation`: 2026-04-13 boundary check 수행
  - `StripesFramework/stripes` `examples`: 2026-04-13 boundary check 수행

## 운영 방식

- 공개 샘플 게이트 결과는 `real_project_validation_gate.md`에 누적 기록한다.
- 합성 fixture로 환원 가능한 실패는 fixture로 내리고, 환원 불가능한 것은 profile boundary 문서로 남긴다.
