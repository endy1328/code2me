# 진입 흐름 조립 구현 메모

## 구현 범위

- `src/adapters/java-source-basic.ts`
- `src/adapters/spring-xml.ts`
- `src/core/report.ts`
- `tests/legacy-java-ee.spec.ts`
- `samples/legacy-java-ee-minimal`

## 반영 내용

1. `java-source-basic`
   - `@RequestMapping` 계열을 읽어 `requestMappings`를 controller/action 메타데이터에 저장
   - class-level/method-level mapping 결합 시 잘못된 중복을 줄이도록 정리
2. `spring-xml`
   - `ViewResolver` 관련 bean property를 artifact로 남겨 후속 해석 근거를 제공
3. `web-xml`
   - `url-pattern`, `servletName`, `contextConfigLocation`을 route/entry metadata로 남김
4. `report`
   - `Primary Entry Flows` 추가
   - `requestMappings + web.xml + DispatcherServlet + Spring 설정 + JSP/Layout` 기반 entry-flow synthesis
   - `요청 URL -> Dispatcher -> Controller -> View -> Layout` 체인 우선 렌더링
   - 관련 `Service/DAO` 요약과 `Entry Flow -> Data Flow` 교차 탐색 추가
   - controller-only, view-only fallback 유지
   - lazy render 및 panel cache 유지

## 남은 구현 포인트

- Spring XML의 handler mapping을 더 직접적으로 읽는 규칙 추가
- 특정 상용 프로젝트 계열의 `Action + ModelAndView` 패턴 복원 강화
- 중복 진입 흐름 정규화
