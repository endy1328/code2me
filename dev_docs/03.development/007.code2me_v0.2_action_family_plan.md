# code2me v0.2 Action-family Plan

## 목적

- `v0.1`에서 고정한 `legacy-java-ee` 기반을 유지하면서 `action-family-legacy-web` 프로파일을 실제 사용 가능한 수준으로 끌어올린다.
- 대상은 Struts / Stripes 계열의 `*.action` 중심 레거시 웹 프로젝트다.
- 판단 기준은 "대표 entry -> action handler -> view/non-screen/result -> supporting evidence"가 리포트에서 추적 가능한가다.

## 현재 완료

- `action-family-legacy-web` 프로파일과 `action-config` 어댑터 추가
- Struts2 `struts.xml` action / method / result JSP 복원
- Stripes `@UrlBinding`, `@DefaultHandler`, `@HandlesEvent` 복원
- Stripes `ForwardResolution`, `StreamingResolution`, self redirect 최소 해석
- Struts `redirectAction` 최소 해석
- Stripes JSP `beanclass` / submit name 기반 역방향 action 링크 복원
- Stripes `session.getAttribute("/actions/*.action")` alias를 `Flow Details > Request Path` supporting evidence로 보존
- Stripes typed session alias인 `session.getAttribute("accountBean")`를 ActionBean 타입 선언/캐스팅에서 route hint로 보존
- request handler 병합 시 `sessionRouteHints` / `redirectActionClasses` 보존
- split report asset 구조에 맞춘 회귀 테스트 정리
- mapper namespace suffix fallback과 screen flow variant grouping 보강

## 남은 구현 작업

### P0. Representative real-project gate 반복

- `mybatis/jpetstore-6`를 최신 `code2me_v0.2`로 재분석한다.
- Stripes session alias evidence가 실제 샘플에서 잡히는지 확인한다.
- 결과를 `dev_docs/04.testing/006.real_project_validation_gate.md`에 기록한다.
- 산출물에서 `screenFlowCards`, `apiFlowCards`, `session route hints`, 주요 redirect target을 확인한다.

### P1. `accountBean` style typed session attribute alias 검증/보강

- status: `done`
- `session.getAttribute("accountBean")`처럼 route 문자열이 아닌 bean-name alias를 검증했다.
- 타입 캐스팅 또는 변수 선언에서 ActionBean 클래스를 추론해 `sessionRouteHints`에 action route 후보를 supporting evidence로 추가한다.
- `jpetstore-6`의 `OrderActionBean.viewOrder`에서 `/Account.action` hint 복원을 확인했다.
- 이 evidence는 flow classification으로 승격하지 않고 request path supporting evidence로만 유지한다.

### P1. Struts redirect-action / chain wildcard 세분화

- `redirectAction`, `chain`, wildcard action result가 실제 route로 복원되는지 확인한다.
- `apache/struts-examples`의 `crud`, `spring-struts`를 우선 검증 대상으로 사용한다.
- wildcard route는 과도한 확장보다 representative route와 supporting evidence 중심으로 처리한다.

### P2. Struts 1 1차 지원

- `struts-config.xml`의 `action path`, `type`, `forward`, `form-bean`을 읽는다.
- action class와 JSP forward를 연결한다.
- 최소 fixture를 먼저 만들고 공개 샘플 검증은 그 뒤에 진행한다.

### P2. `DynamicMappingFilter` boundary 정의

- 동적 mapping filter 계열을 현재 프로파일에서 어디까지 흡수할지 정한다.
- 충분한 evidence가 없으면 no-match 또는 low-confidence supporting evidence로 둔다.
- 구현 전 fixture와 문서 기준을 먼저 만든다.

### P2. read-only gate 반복

- 0.2 변경 후에도 target `.code2me` 쓰기 실패 시 internal mirror가 정상 생성되는지 확인한다.
- `action-family-legacy-web` 샘플과 대표 공개 샘플 각각에서 최소 1회 확인한다.

- Struts / Stripes 최소 fixture가 모두 통과한다.
- 대표 공개 샘플에서 `action-family-legacy-web` 프로파일이 선택된다.
