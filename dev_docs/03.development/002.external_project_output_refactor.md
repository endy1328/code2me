# 외부 프로젝트 출력 경로 리팩터링 이력

## 변경 범위

- `src/core/store.ts`
- `src/core/analysis.ts`
- `src/cli/index.ts`
- `tests/legacy-java-ee.spec.ts`
- `README.md`

## 당시 구현 내용

1. 출력 경로 계산을 `getOutputPaths(projectRoot)`로 분리
2. 저장 위치를 `.code2me-data/projects/<project-key>/`로 변경
3. `analyzeProject`가 `snapshot`과 `outputPaths`를 함께 반환하도록 수정
4. CLI JSON 출력에 `sourceRoot`, `outputDir`, `reportPath`, `summaryPath` 포함
5. 테스트를 내부 저장소 기준으로 갱신

## 현재 상태

- 현재 구현은 사용자 요청에 따라 다시 `대상 프로젝트 아래 .code2me/` 저장 구조를 사용한다.
- 현재 화면 구성은 `Entry Flow` 중심 리포트로 유지되고, 저장 경로와 UI 흐름은 별도 축으로 관리한다.
- 이 문서는 외부 프로젝트 출력 경로를 내부 저장소로 바꾸었던 작업 이력으로 유지한다.
