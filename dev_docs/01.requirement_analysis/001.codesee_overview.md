# CodeSee 조사 개요

## 문서 목적

이 문서는 CodeSee가 어떤 제품인지, 어떤 문제를 풀고 어떤 방식으로 가치를 제공하는지 빠르게 파악하기 위한 개요 문서다. 조사 기준 시점은 2026-04-06(KST)다.

## 한 줄 정의

CodeSee는 코드베이스를 자동으로 시각화하고, PR 리뷰와 서비스 구조 이해를 돕는 코드 가시성(code visibility) 제품이다. 핵심은 "코드를 읽기 전에 코드의 구조와 영향 범위를 먼저 보게 해주는 것"이다.

## 현재 상태

- `codesee.io` 공식 사이트와 `docs.codesee.io` 공식 문서는 2026-04-06 기준 접근 가능했다.
- GitKraken 공식 블로그는 CodeSee 인수 발표 날짜를 2024-05-14로 명시한다.
- 따라서 현재 CodeSee는 독립 브랜드 흔적을 유지하면서도 GitKraken DevEx 플랫폼과 기술적으로 결합된 상태로 보는 것이 가장 안전하다.
- 이 마지막 해석은 공식 발표와 현재 사이트 상태를 바탕으로 한 추론이다.

## CodeSee가 해결하려는 문제

CodeSee가 반복적으로 겨냥하는 문제는 아래와 같다.

- 대형 코드베이스에서 구조를 머릿속으로만 이해해야 하는 부담
- 신규 입사자, 팀 이동자, 장애 대응 인력이 낯선 코드를 빠르게 이해해야 하는 상황
- PR 리뷰가 파일 이름 순서와 diff 중심으로 진행되어 영향 범위를 놓치기 쉬운 문제
- 서비스, 저장소, 함수, 데이터 흐름 간 의존 관계를 문서 대신 암묵지에 의존하는 문제
- 변경 영향 분석이 늦어져 버그, 리뷰 지연, 온보딩 비용이 커지는 문제

## 제품 포지셔닝

CodeSee는 정적 문서 도구도 아니고 일반적인 코드 검색 도구도 아니다. 공식 메시지를 기준으로 보면 아래 세 가지를 결합한 제품이다.

1. 코드 구조 시각화 도구
2. 리뷰 워크플로우 보조 도구
3. 코드 지식 축적 및 공유 도구

즉, "코드 이해"를 맵 기반 인터페이스와 협업 기능으로 운영 워크플로우에 끼워 넣는 제품이다.

## 핵심 제품 축

공식 사이트와 문서에서 반복적으로 드러나는 핵심 축은 다음과 같다.

- Codebase Maps: 저장소와 파일/디렉터리/의존 관계 시각화
- Review Maps: PR 변경분을 코드베이스 맥락 안에서 보는 시각 리뷰
- Tours: PR 또는 코드 이해를 위해 순서를 지정한 안내 지점
- Labels / Notes / Comments: 지도 위에 지식과 문맥을 고정하는 기능
- Service Maps: APM/트레이싱 데이터와 저장소를 연결한 서비스 구조 시각화
- Flows / Variants: 서비스 간 데이터 흐름과 경로 차이를 보는 기능
- Code Automations: 규칙, 알림, 지식 강제 적용 자동화
- AI: 코드/PR/서비스 요약, 리뷰 투어 생성 보조

## 핵심 가치 제안

공식 메시지를 정리하면 CodeSee의 가치 제안은 아래 네 줄로 압축된다.

- 코드 변경 전에 구조를 먼저 이해하게 한다.
- 리뷰를 diff 나열이 아니라 영향 중심 탐색으로 바꾼다.
- 개인의 머릿속 지식을 맵 위의 공유 자산으로 바꾼다.
- 대규모/레거시 코드베이스에서 온보딩과 변경 리스크를 낮춘다.

## 주요 사용자

공식 콘텐츠 기준으로 예상되는 주요 사용자는 다음과 같다.

- 신규 합류자 또는 팀 이동 개발자
- 대형 PR을 자주 다루는 개발팀
- 여러 저장소/서비스를 동시에 운영하는 팀 리드 및 플랫폼 팀
- 레거시 시스템을 유지보수하거나 리팩터링하는 엔지니어
- 장애 대응 시 특정 경로와 연관 코드를 빨리 찾고 싶은 운영/개발 인력

## 사용 흐름 요약

CodeSee의 일반적인 가치 전달 흐름은 대체로 아래와 같다.

1. 저장소 또는 서비스 데이터를 연결한다.
2. 코드/서비스 맵을 자동 생성한다.
3. 사용자는 맵 위에서 영향 범위, 연결 관계, 숨은 의존성을 탐색한다.
4. 필요 시 라벨, 노트, 코멘트, 투어로 팀 지식을 덧입힌다.
5. PR 리뷰, 온보딩, 리팩터링, 장애 대응 등 실무 흐름에 재사용한다.

## 강점

- 가치 제안이 매우 구체적이다. "코드 이해"라는 넓은 문제를 맵 기반 경험으로 좁혔다.
- 자동 생성과 자동 업데이트를 내세워 문서 부패 문제를 회피하려 한다.
- 단일 저장소뿐 아니라 서비스/멀티리포 맥락까지 확장한다.
- 리뷰, 온보딩, 아키텍처 이해, 협업 기록을 한 제품군으로 엮었다.
- AI를 그래프/맵 위에 얹어 요약과 설명 도구로 쓴다.

## 한계 및 주의점

- 시각화 품질은 분석 정확도와 스케일링 품질에 강하게 의존한다.
- 복잡한 저장소일수록 자동 추론만으로는 정확도가 부족해 설정 파일이나 수동 보정이 필요하다.
- 시각화는 강력하지만, 잘못 그려진 맵은 잘못된 확신을 줄 수 있다.
- 협업 가치가 큰 대신 개인 개발자용 경량 도구보다는 팀 도입형 제품 성격이 강하다.
- 서비스 맵/AI/자동화 등 고부가 기능은 상위 티어 제약이 있어 보인다.

## 비슷한 제품을 만들 때 바로 참고할 포인트

- CodeSee의 진짜 차별점은 "그래프를 그린다"가 아니라 "개발 워크플로우에 맞는 시각적 의사결정 인터페이스"를 제공한다는 점이다.
- 단순 AST 시각화 도구만으로는 CodeSee와 비슷한 체감을 만들기 어렵다.
- 최소한 `자동 생성`, `영향 분석`, `리뷰 맥락화`, `지식 주석`, `지속 업데이트` 다섯 축이 함께 있어야 유사 제품으로 인식될 가능성이 높다.
