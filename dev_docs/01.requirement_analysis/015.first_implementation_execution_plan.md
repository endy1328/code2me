# 첫 구현 실행 계획 초안

## 문서 목적

이 문서는 code2me의 첫 구현을 `Legacy Java EE 프로파일 기반 온보딩 분석기 MVP`로 고정하고, 실제 개발 순서와 작업 분해를 정의하기 위한 실행 계획 문서다.

## 첫 구현 목표

첫 구현의 목표는 아래와 같다.

- 범용 분석기 코어의 최소 구조를 만든다.
- `Legacy Java EE 프로파일`을 첫 실전 프로파일로 구현한다.
- 최소 어댑터 4종으로 구조, 진입점, 핵심 흐름을 추출한다.
- 병합 로직으로 하나의 탐색 가능한 결과 그래프를 만든다.

## 첫 구현 범위

### 포함

- 공통 메타모델 최소 구현
- 분석 프로파일 공통 구조 구현
- 어댑터 인터페이스와 레지스트리 구현
- `Legacy Java EE 프로파일` 구현
- 최소 어댑터 4종 구현
  - `ant-build-xml`
  - `web-xml`
  - `spring-xml`
  - `java-source-basic`
- 결과 병합 로직 최소 구현

### 후순위

- `ibatis-sql-map`
- `mybatis-mapper`
- `jsp-view`
- `sitemesh-config`
- 프로파일 자동 추천 고도화
- 증분 분석

### 제외

- 복수 프로파일 동시 고도화
- 정밀 메서드 수준 call graph
- 완전한 Spring bean wiring 재현
- 모든 운영환경 변종 정밀 해석

## 권장 실행 순서

1. 공통 코어 타입 확정
2. 어댑터 인터페이스와 레지스트리 구현
3. Legacy Java EE 프로파일 고정
4. 최소 어댑터 4종 구현
5. 결과 병합 로직 구현
6. 검증 시나리오 적용
7. 확장 어댑터 추가

## 병렬 처리 후보

- 공통 코어 타입과 프로파일 타입 문서화
- 어댑터 인터페이스와 레지스트리 설계
- Legacy Java EE 프로파일 명세 보정
- 검증 시나리오 문서화

단, 최소 어댑터 구현 이후의 병합 로직은 앞선 결과에 의존하므로 메인 조율 아래 순차 처리하는 편이 안전하다.

## 단계별 성공 기준

### Phase 1

- 코어 타입, 프로파일 타입, 어댑터 인터페이스가 구현된다.

### Phase 2

- 최소 어댑터 4종으로 대표 구조가 추출된다.

### Phase 3

- 병합 후 하나의 결과 그래프가 생성된다.

### Phase 4

- 샘플 프로젝트 기준 검증을 통과한다.

### Phase 5

- 확장 어댑터로 persistence/view 흐름까지 이어진다.

## 최종 완료 기준

- 공통 코어가 Legacy Java EE와 향후 다른 프로파일을 수용할 수 있다.
- 최소 어댑터 4종만으로 실제 분석 결과가 생성된다.
- 병합 후 중복과 추론 충돌이 제어된다.
- 검증 기준이 문서화되어 다음 구현 단계로 바로 이어질 수 있다.
