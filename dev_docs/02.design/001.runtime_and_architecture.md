# 런타임 및 아키텍처 결정

## 결정

- 구현 언어/런타임: `TypeScript + Node.js`
- 앱 형태: `CLI-first, library-backed`
- 초기 저장 방식: `.code2me/` 아래 JSON 스냅샷과 이력 파일
- 첫 수직 슬라이스: `Legacy Java EE` 최소 어댑터 4종
- 현재 리포트: `시작점 / Start Here`, `진입 흐름 / Entry Flow`, `데이터 흐름 / Data Flow`, `구조 / Structure`

## 이유

- 현재 설계는 프로파일/어댑터/메타모델 분리가 핵심이라 정적 타입 이점이 크다.
- Node.js 환경은 파일 스캔과 CLI 조립 속도가 빠르다.
- 초기에는 웹 UI나 DB보다 분석 코어 검증이 우선이다.
- 레거시 Java 진입 흐름은 `web.xml -> Spring XML -> Controller/Action -> View -> Layout` 순으로 해석하는 것이 사용성에 유리하다.

## 초기 코드 구조

- `src/core`: 메타모델, 프로파일, 어댑터, 실행기
- `src/scanner`: 파일 인덱싱
- `src/profiles`: 프로파일 정의
- `src/adapters`: 파일 유형별 어댑터
- `src/merge`: 병합 로직
- `src/cli`: CLI 진입점
- `tests`: fixture 기반 테스트
- `samples`: 샘플 입력 저장소
