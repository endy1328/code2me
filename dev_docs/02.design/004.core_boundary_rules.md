# 코어 경계 규칙

## 목적

- 현재 `Legacy Java EE` 구현을 더 깊게 파면서도, 이후 `범용 분석기`로 확장 가능한 최소 경계를 고정한다.
- 경계를 너무 일찍 과도하게 일반화하지 않고, 이미 검증된 반복 패턴만 코어 책임으로 남긴다.

## 현재 원칙

- 현재 제품의 1차 목표는 `Legacy Java EE` 분석 품질 고정이다.
- 공통 코어는 “어떤 프로파일에도 공통으로 유지되어야 하는 의미”만 가진다.
- 프로파일 전용 규칙은 adapter 또는 report 해석 단계에 남기고, 코어로 섣불리 끌어올리지 않는다.

## 경계 구분

### 1. Profile

`profile`은 어떤 프로젝트를 어떤 전략으로 읽을지를 결정한다.

현재 책임

- 프로젝트 타입 판별
- detection score / reasons 계산
- 필요한 adapter 묶음 선언

현재 비책임

- 실제 파일 파싱
- 구체적인 merge 규칙
- 리포트 렌더링 세부 규칙

## Merge 규칙

merge는 “자동 확정”보다 “근거 보존”을 우선한다.

현재 원칙

- 같은 node id / edge id로 들어온 결과만 우선 병합한다.
- `sourceAdapterIds`, `evidence`, `tags`는 누적 보존한다.
- 더 구체적인 path와 더 강한 confidence가 있으면 제한적으로 상향 반영한다.
- `requestMappings`, `requestHandlers`는 유실 없이 합쳐야 한다.
- 직접 근거가 있는 결과가 fallback보다 우선한다.

코어에 남아야 할 merge 규칙

- 중복 제거
- 근거 누적
- confidence 보수적 상향
- handler 메타데이터 유실 방지

프로파일/도메인 전용으로 남겨야 할 merge 규칙

- XML mapping 우선순위
- mapper fallback 허용 범위
- SiteMesh alias/direct 세부 판단

현재 관련 코드

- [`src/merge/merge.ts`](/home/u24/projects/code2me/src/merge/merge.ts)

## Confidence 규칙

코어는 confidence 자체의 저장 구조와 최소 정책만 가진다.

공통 규칙

- `high`: 직접 식별자 또는 직접 근거가 있는 경우
- `medium`: 구조적 연결은 있으나 세부 근거가 덜 강한 경우
- `low`: fallback 또는 휴리스틱 중심인 경우

도메인 전용 규칙

- `confirmed / inferred / heuristic` 분류
- `heuristic` 기본 숨김
- SQL fallback / name stem fallback

현재 관련 문서

- [docs/inferred-data-path-policy.md](/home/u24/projects/code2me/docs/inferred-data-path-policy.md)

## Report 경계

report는 snapshot을 사용자 관점 흐름으로 재조립하는 계층이다.

현재 책임

- framework flow, screen flow, non-screen flow 합성
- business/data/context 설명 조립
- primary persona 기준 정보 우선순위 적용

현재 비책임

- 원시 파싱
- adapter 내부 추론 규칙 대체
- 직접 근거가 없는 데이터를 확정 사실로 승격

## 다음 프로파일 확장 시 지켜야 할 규칙

### 코어로 올려도 되는 것

- 새로운 엔터티/관계가 여러 프로파일에서 반복될 때
- confidence 저장 구조가 공통일 때
- merge 원칙이 프로파일 독립적일 때

### 코어로 올리면 안 되는 것

- 특정 XML 프레임워크 이름
- 특정 naming fallback 규칙
- 특정 리포트 문구와 휴리스틱
- 아직 하나의 프로파일에서만 검증된 예외 처리

## 현재 다음 단계

- `Legacy Java EE`에서 반복적으로 검증된 규칙만 코어로 남긴다.
- 다음 프로파일은 구현보다 `입력 신호 / 필요한 엔터티 / 필요한 관계` 문서화부터 시작한다.
- 코어 변경은 항상 `fixture + regression`이 먼저 있어야 한다.
