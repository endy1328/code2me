# 내부 출력 저장 구조 검토 이력

## 검토 내용

- 분석 결과를 원본 프로젝트 루트가 아니라 `code2me/.code2me-data/projects/<project-key>/` 아래에 저장하는 안을 검토했다.
- `project-key`는 `basename + sourceRoot hash` 조합으로 만드는 방식을 사용했다.
- `snapshot.json`, `history.jsonl`, `summary.md`, `report.html`를 내부 저장소에 기록하는 구조를 시험했다.

## 검토 이유

- 기존 요구사항의 `메타데이터 저장: 프로그램 내부 저장`, `원본 소스: 절대 수정하지 않음`을 지켜야 한다.
- 실제 분석 대상은 읽기 전용 저장소이거나 외부 경로일 수 있다.
- 같은 이름의 서로 다른 프로젝트를 충돌 없이 보관해야 한다.

## 현재 상태

- 현재 구현은 사용자 요청에 따라 다시 `대상 프로젝트 아래 .code2me/` 저장 방식으로 운용한다.
- 현재 리포트 UI는 저장 위치와 별개로 `Start Here`와 `Entry Flow` 중심으로 보여준다.
- 이 문서는 내부 저장 구조를 검토했던 이력으로 남긴다.
